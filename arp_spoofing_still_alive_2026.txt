# Executive Summary

This study investigates **Address Resolution Protocol (ARP) Spoofing attacks** using a **network forensic approach based on the National Institute of Justice (NIJ) methodology**.

The authors aim to identify:

* The attacker's MAC address
* The victim's MAC address
* The exact attack timestamp

The study combines:

* Controlled ARP spoofing simulations
* Packet capture analysis
* Traffic correlation
* Log analysis

Using **Wireshark** and **XARP**, the researchers demonstrate that the NIJ framework can effectively support forensic investigations by producing structured and legally defensible evidence.

A key finding is that **XARP detected the attack 13 seconds earlier than Wireshark**, suggesting the value of dedicated ARP monitoring tools in operational environments.

---

# Research Objectives

The study seeks to:

1. Detect ARP spoofing attacks in a controlled network.
2. Identify attackers and victims using IP and MAC address correlation.
3. Determine attack timestamps accurately.
4. Evaluate the effectiveness of the NIJ forensic methodology.
5. Generate legally defensible digital evidence.

---

# Background

ARP converts IP addresses into MAC addresses within local networks.

Because ARP requests are broadcast and lack authentication mechanisms, attackers can exploit the protocol by sending forged ARP replies.

Successful ARP spoofing attacks enable:

* Man-in-the-Middle (MITM) attacks
* Denial-of-Service (DoS) attacks
* Traffic interception
* Data manipulation
* Credential theft

---

# Why This Research Matters

As internet usage continues to grow, organizations increasingly rely on networked systems.

However, many networks remain vulnerable to ARP-based attacks because:

* ARP lacks built-in security controls.
* Devices trust unsolicited ARP responses.
* Attack detection mechanisms are often reactive.

The study emphasizes that forensic readiness is essential for both:

* Incident response
* Legal investigation

# Attack Scenario

The attacker (PC 2) used **Ettercap** to launch an ARP spoofing attack against:

* RouterBoard
* PC 1

The attacker injected forged ARP responses claiming ownership of the router's IP address.

As a result:

* Victim devices updated their ARP caches incorrectly.
* Traffic intended for the router was redirected to the attacker.
* The attacker positioned itself between communicating devices.

# Key Findings

## Detection Results

The attack was successfully detected using both tools.

### First Detection Time

| Tool      | Detection Timestamp       |
| --------- | ------------------------- |
| XARP      | 20 January 2024, 16:25:30 |
| Wireshark | 20 January 2024, 16:25:43 |

### Observation

XARP detected the attack **13 seconds earlier** than Wireshark.

# Practical Takeaways for Researchers

## 1. Dedicated ARP monitoring tools matter

XARP provided earlier detection than general packet analysis tools.

## 2. Time correlation is critical

Combining timestamps from multiple sources improves forensic accuracy.

## 3. Packet capture alone is insufficient

Integrating:

* Logs
* Traffic correlation
* Protocol analysis

produces stronger evidence.

## 4. Forensic frameworks improve reproducibility

The NIJ methodology provides a repeatable investigation process.

---

# Limitations

The study has several limitations:

* Small-scale network environment
* Single attacker scenario
* Limited device diversity
* No encrypted traffic analysis
* No large-scale enterprise validation
