# Summary of Presentation

## Title
Defining Micro-Sandbox Boundaries for AI  
數十億個沙盒，打造更智慧、更安全的 AI  
(Translation: Hundreds of Millions of Sandboxes, Creating Smarter and Safer AI)

## Author/Presenter
Melody Huang  
Sr. Solutions Architect, Cloudflare

## Key Points

- **AI Code Generation Growth**  
  AI is increasingly responsible for code generation, with major companies like Microsoft and Google generating over 30% of code through AI. Cursor generates over 1 billion lines of code daily, indicating an explosion in AI-developed software.

- **Challenges of AI Code Quality**  
  There is a rising concern about "who wrote the bad code?" as AI-generated code begins to proliferate, underscoring the need for better oversight and quality control.

- **Scalability Issues**  
  With many AI agents or bots operating simultaneously, scalability and management become critical challenges.

- **Security Concerns**  
  Highlighted vulnerabilities include file exfiltration and unauthorized access due to weak or flawed sandbox isolation, as demonstrated with examples of attacks on Claude Cowork.

- **Sandboxing using Cloudflare Workers**  
  Cloudflare Workers are used to create isolated micro-sandboxes that allow for better security and scalability by compartmentalizing AI worker environments.

- **Dynamic Workers**  
  Dynamic workers enable deployment of sandboxed, serverless functions generated by AI with controlled permissions and no global internet access to improve security.

- **Limiting Data Exposure**  
  Proposal to limit AI agents' access to secrets (e.g., sensitive email data) through controlled toolboxes and proxies, minimizing data leakage risks.

- **Multi-layered Execution Ladder for AI Agents**  
  A scalable architecture that starts from a minimal layer (Tier 0) and adds capabilities as needed through five computational levels: Workspace, Dynamic Worker, Worker + npm, Browser, and Sandbox.

- **AI Agent Evolution in Three Waves**  
  1. Chatbots - Stateless, reactive, Q&A style  
  2. Coding Agents - Stateful, with tools, code reading/writing/execution, single-user mode  
  3. Autonomous Agents - Long-running, distributed orchestration, fleets of agents, personalized per user/task/email

- **Capabilities of Advanced Agents**  
  Ability to think, converse (voice and text), observe (complete browser access), recover from crashes, create child agents, and persist memory through advanced techniques.

## Important Conclusions and Insights

- AI systems' rapid growth necessitates a novel approach to AI worker deployment and management.
- Micro-sandboxes with strong isolation are critical to ensure security and scalability when running autonomous AI agents.
- Securing sensitive data and reducing attack surfaces require restricting AI agents' direct access to secrets and using proxies or toolboxes for controlled interaction.
- The future of AI agents involves autonomous, distributed fleets that can handle complex workflows and maintain long-term memory and resilience.
- Cloudflare’s infrastructure and technologies like Dynamic Workers are well-positioned to handle the operational and security challenges posed by this evolution in AI agent capabilities.

---

## Technical and Other Problems Emphasized

- **Code Quality**: Increasing volume of AI-generated code raises concerns about buggy or insecure code entering production.
- **Scalability**: Managing potentially millions to billions of AI workers/agents requires efficient scaling mechanisms; single-use bots are not sustainable.
- **Security Flaws**: Existing isolation mechanisms can be bypassed, enabling attackers to exfiltrate sensitive user files and secrets.
- **Data Leakage Risk**: Email agents and others accessing sensitive data can inadvertently expose confidential information.
- **Sandbox Isolation Challenges**: Ensuring true isolation in a cloudworker environment remains complex and requires new APIs and architectural models.
- **Network Access Control**: Preventing AI agents from unauthorized internet access ("globalOutbound" permissions) is vital to limit attack or data leakage vectors.
- **Agent Bypass Vulnerabilities**: Malicious code or faulty agent design may lead to bypassing security mechanisms, accessing secrets or unauthorized email data.
- **Persistent State Management**: Designing agents that maintain memory and recover from failures requires sophisticated checkpointing and state isolation.
  
---

This presentation stresses the importance of comprehensive sandboxing, granular permissioning, and layered agent design to foster a future AI ecosystem that is intelligent, secure, and scalable.
